% 1. OBJECTIVES
% -----------------------------------------------
\section{Objectives}

The goal of this project is to design and develop a server-side Web application called
\textbf{Lecture Notes App}, a platform that allows university students to upload, share,
search, and download lecture notes in PDF format.
Users can register and log in to the platform, organize notes by course, mark notes
as favourites, and rate content shared by other students.
The application is built using Java Servlets, JSP pages, and a PostgreSQL database,
deployed on Apache Tomcat.

% -----------------------------------------------
% 2. MAIN FUNCTIONALITIES
% -----------------------------------------------
\section{Main Functionalities}

The application provides the following main functionalities:

\begin{itemize}
    \item \textbf{User registration and authentication:} users can create an account
          (\texttt{RegisterServlet}), log in (\texttt{LoginServlet}),
          log out (\texttt{LogoutServlet}), and view/edit their profile
          (\texttt{ProfileServlet}).

    \item \textbf{Note management:} authenticated users can upload PDF notes
          associated with a course (\texttt{UploadNoteServlet}), update note metadata
          (\texttt{UpdateNoteServlet}), delete their own notes
          (\texttt{DeleteNoteServlet}), and download notes uploaded by others
          (\texttt{DownloadNoteServlet}).

    \item \textbf{Note search:} users can search notes by title, course, or author
          (\texttt{SearchNoteServlet}).

    \item \textbf{Favourites:} users can add notes to a personal favourites list
          and view them on a dedicated page (\texttt{favorites.jsp}).

    \item \textbf{Ratings:} users can rate notes uploaded by other students,
          contributing to a quality score visible on the home page.

    \item \textbf{Course overview:} a course summary view aggregates the notes
          available for each course (\texttt{CourseSummaryServlet}).
\end{itemize}

% 3. DATA LOGIC LAYER

\section{Data Logic Layer}

\subsection{Entity-Relationship Schema}

The ER schema models the following main entities: \texttt{User}, \texttt{Note},
\texttt{Course}, \texttt{Favorite}, and \texttt{Rating}.
A \texttt{User} can upload many \texttt{Note}s; each \texttt{Note} belongs to
exactly one \texttt{Course}; a \texttt{User} can mark many \texttt{Note}s as
\texttt{Favorite}; a \texttt{User} can assign one \texttt{Rating} per \texttt{Note}.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.95\textwidth]{../../diagrams/ERdiagram.drawio.png}
    \caption{Entity-Relationship Schema of the Lecture Notes App}
    \label{fig:er-schema}
\end{figure}

\subsection{Other Information}

The \texttt{Note} entity stores the following attributes: \texttt{id},
\texttt{authorId}, \texttt{authorUsername}, \texttt{courseId}, \texttt{courseName},
\texttt{title}, \texttt{description}, \texttt{uploadDate}, and \texttt{filePath}
(path to the PDF stored on the server or on AWS S3, as indicated by the
\texttt{software.amazon.awssdk:s3} dependency in the POM).

The \texttt{User} entity stores login credentials and profile information.
The \texttt{Rating} entity links a user to a note with a numeric score.
The \texttt{Favorite} entity represents the many-to-many association between
users and their favourite notes.

% -----------------------------------------------
% 4. PRESENTATION LOGIC LAYER
% -----------------------------------------------
\section{Presentation Logic Layer}

The presentation layer is implemented with JSP pages located in
\texttt{src/main/webapp/jsp/}.
The application entry point is \texttt{login.jsp}, as declared in \texttt{web.xml}
as the welcome file.

\subsection{Login Page -- \texttt{login.jsp}}
Presents a form for user authentication (username and password).
On success the user is redirected to the home page.

\subsection{Register Page -- \texttt{register.jsp}}
Allows new users to create an account by providing username, email, and password.

\subsection{Home Page -- \texttt{home.jsp}}
The main dashboard after login. Displays the list of available notes with
search and filter controls. Users can browse by course, download notes,
or add them to favourites directly from this page.

\subsection{Upload Page -- \texttt{upload.jsp}}
Form for uploading a new note: the user selects the course, inserts a title
and description, and attaches a PDF file.

\subsection{Profile Page -- \texttt{profile.jsp}}
Shows the user's personal information and the notes they have uploaded.
Allows updating profile data and deleting own notes.

\subsection{Favourites Page -- \texttt{favorites.jsp}}
Lists all notes that the logged-in user has marked as favourite.

\textbf{Controller} (Servlets), \textbf{Model} (Plain Java Objects), and
\textbf{DAO} (data-access objects interfacing with PostgreSQL via JDBC).

The typical interaction for note upload proceeds as follows:
the browser sends a \texttt{POST multipart/form-data} request to
\texttt{/upload}; \texttt{UploadNoteServlet} validates the session,
extracts the file and metadata, invokes the DAO to persist the record,
stores the file (locally or on AWS S3), and redirects to the home page.

\subsection{REST API Details}

\subsubsection{Upload Note -- \texttt{POST /upload}}

\begin{description}
    \item[URL] \texttt{/upload}
    \item[Method] \texttt{POST} (\texttt{multipart/form-data})
    \item[URL Parameters] None
    \item[Data Parameters] \texttt{title} (string), \texttt{description} (string),
          \texttt{courseId} (int), \texttt{file} (PDF, max 10\,MB)
    \item[Success Response] \texttt{302 Found} -- redirect to \texttt{/home}
    \item[Error Response] \texttt{400} if fields are missing;
          \texttt{401} if session is invalid; \texttt{500} on DB/S3 error
\end{description}

\subsubsection{Search Notes -- \texttt{GET /note/search}}

\begin{description}
    \item[URL] \texttt{/note/search}
    \item[Method] \texttt{GET}
    \item[URL Parameters] \texttt{keyword} (optional), \texttt{courseId} (optional)
    \item[Data Parameters] None
    \item[Success Response] \texttt{200 OK} -- forwards to \texttt{home.jsp} with
          filtered note list
    \item[Error Response] \texttt{401} if not authenticated; \texttt{500} on DB error
\end{description}

\subsubsection{Download Note -- \texttt{GET /note/download}}

\begin{description}
    \item[URL] \texttt{/note/download}
    \item[Method] \texttt{GET}
    \item[URL Parameters] \texttt{noteId} (int, required)
    \item[Data Parameters] None
    \item[Success Response] \texttt{200 OK} -- streams PDF with
          \texttt{application/pdf} content type
    \item[Error Response] \texttt{404} if note does not exist;
          \texttt{401} if not authenticated
\end{description}

\subsubsection{Delete Note -- \texttt{POST /note/delete}}

\begin{description}
    \item[URL] \texttt{/note/delete}
    \item[Method] \texttt{POST}
    \item[URL Parameters] None
    \item[Data Parameters] \texttt{noteId} (int)
    \item[Success Response] \texttt{302 Found} -- redirect to \texttt{/profile}
    \item[Error Response] \texttt{403} if the requesting user is not the author;
          \texttt{404} if note does not exist
\end{description}
